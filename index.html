📌 What is a .exe File?

📦

Executable File

A file that runs directly on Windows without any additional software

🎯

Why Convert Python to .exe?

✓ Share with people who don't have Python
✓ No installation needed on their computer
✓ Professional distribution
✓ Protects your code (partially)

✨ Real Example: You create a calculator in Python. Convert to .exe → Anyone double-clicks and it runs!

🛠️ What You Need Before Starting

🐍

Python

Already installed on your computer

Check: Open Command Prompt and type python --version

📦

pip

Comes with Python - used to install packages

Check: pip --version

⚠️ Important: Make sure Python is installed and working before continuing!

📦 Step 1: Install PyInstaller

1 Open Command Prompt

• Press Windows key on keyboard

• Type cmd

• Click on "Command Prompt"

2 Type this command and press Enter:

pip install pyinstaller

3 Wait for installation to complete

You should see "Successfully installed pyinstaller"

💡 Tip: If you see an error, try: python -m pip install pyinstaller

📝 Step 2: Create a Python Script to Convert

1 Open any text editor (Notepad, VS Code, etc.)

2 Type this code:

# save as: myprogram.py

print("=" * 40)
print("   Welcome to My Python Program!")
print("=" * 40)

name = input("Enter your name: ")
print(f"\nHello {name}! Nice to meet you!")

print("\nPress Enter to exit...")
input()

3 Save the file as myprogram.py

Save it somewhere easy to find, like your Desktop

💡 Tip: First test your .py file by double-clicking it to make sure it works!

💻 Step 3: Open Command Prompt in Your Script Folder

1 Go to the folder where you saved myprogram.py

2 Click on the address bar at the top of the folder window

Example: C:\Users\YourName\Desktop

3 Type cmd and press Enter

Command Prompt will open in that folder!

✅ Result: Command Prompt opens showing your folder path

⚙️ Step 4: Run PyInstaller

1 In Command Prompt, type this command:

pyinstaller --onefile myprogram.py

2 Press Enter

3 Wait for PyInstaller to finish

You will see many lines of text - this is normal!

Look for "Building EXE from EXE-00 completed successfully"

💡 What --onefile does: Creates a SINGLE .exe file (easier to share)

🔄 What Happens While You Wait?

1️⃣ PyInstaller reads your Python script

2️⃣ Finds all imports and dependencies

3️⃣ Bundles Python interpreter + your code together

4️⃣ Creates a single .exe file

⏱️ Time: Takes 10-30 seconds depending on script size
⚠️ Don't worry about: Warnings about missing modules - they usually don't affect your program

📂 Step 5: Find Your .exe File

1 Look in your original folder

2 You will see new folders created:

your_folder/
├── myprogram.py          (your original script)
├── build/                (temporary files - can delete)
├── dist/                 ⭐ YOUR .EXE IS HERE!
│   └── myprogram.exe     ⭐ Double-click to run!
└── myprogram.spec        (configuration file - keep it)
✅ Success! Your .exe file is inside the dist folder

🧪 Step 6: Test Your .exe File

1 Open the dist folder

2 Double-click myprogram.exe

3 Your program should run!

⚠️ Antivirus Warning:
Some antivirus programs may show a warning for new .exe files. This is normal! Click "Run Anyway" or allow the file.
💡 Test on another computer: Copy the .exe to a computer without Python to verify it works!

🧮 Complete Example: Calculator Program

Create a file called calculator.py with this code:

# calculator.py - A simple calculator

def add(x, y): 
    return x + y

def subtract(x, y): 
    return x - y

def multiply(x, y): 
    return x * y

def divide(x, y): 
    if y != 0:
        return x / y
    else:
        return "Error! Cannot divide by zero"

print("=" * 30)
print("     SIMPLE CALCULATOR")
print("=" * 30)
print("1. Add")
print("2. Subtract")
print("3. Multiply")
print("4. Divide")

choice = input("Choose (1-4): ")
num1 = float(input("Enter first number: "))
num2 = float(input("Enter second number: "))

if choice == '1':
    print(f"Result: {add(num1, num2)}")
elif choice == '2':
    print(f"Result: {subtract(num1, num2)}")
elif choice == '3':
    print(f"Result: {multiply(num1, num2)}")
elif choice == '4':
    print(f"Result: {divide(num1, num2)}")
else:
    print("Invalid choice!")

input("\nPress Enter to exit...")

Convert to .exe:

pyinstaller --onefile calculator.py

⚙️ Useful PyInstaller Options

Command What it does When to use
--onefile Creates single .exe file Always use this for sharing
--windowed No console/terminal window For GUI apps (Tkinter, PyQt)
--noconsole Hides the black window Same as --windowed
--icon=file.ico Adds custom icon to .exe To make your app look professional
--name="AppName" Changes output file name To give your .exe a custom name

🪟 Option: Hide the Console Window

For programs with a graphical interface (Tkinter, PyQt, etc.), use --windowed:

pyinstaller --onefile --windowed my_gui_app.py

Example GUI program (save as gui_app.py):

import tkinter as tk

root = tk.Tk()
root.title("My Application")
root.geometry("300x200")

label = tk.Label(root, text="Hello World!", font=("Arial", 16))
label.pack(pady=50)

button = tk.Button(root, text="Click Me", command=root.destroy)
button.pack()

root.mainloop()
💡 Result: No black console window appears - just your GUI application!

🎨 Option: Add a Custom Icon

1 Get or create an .ico file

• Download from: icons8.com, flaticon.com

• Convert PNG to ICO using online converter

2 Place the .ico file in your script folder

Example: myicon.ico

3 Run PyInstaller with the icon option:

pyinstaller --onefile --icon=myicon.ico myprogram.py
✅ Result: Your .exe file will show your custom icon instead of the default Python icon!

📝 Option: Change the .exe Name

Use --name to give your .exe a custom name:

pyinstaller --onefile --name="MyCoolApp" myprogram.py

This creates: MyCoolApp.exe instead of myprogram.exe

💡 Tip: Use a descriptive name that tells people what your app does!

🔧 Combining Multiple Options

You can use multiple options together:

pyinstaller --onefile --windowed --icon=myicon.ico --name="MyApp" myprogram.py
📦

--onefile

Single file output

🪟

--windowed

No console window

🎨

--icon

Custom icon

📝

--name

Custom filename

✅ Result: A professional-looking standalone application!

📊 Why is the .exe File Large?

🐍

Python Interpreter

~5-10 MB

📚

Libraries/Modules

~5-20 MB

📝

Your Code

~1-5 KB (very small!)

📦

Total Size

10-30 MB on average

💡 This is normal! A simple "Hello World" .exe is about 5-7MB because it includes Python itself.

⚠️ Common Error #1: "pyinstaller not recognized"

What you see:

'pyinstaller' is not recognized as an internal or external command

Solution: Try these commands one by one:

python -m pip install pyinstaller
python3 -m pip install pyinstaller
pip3 install pyinstaller
💡 After installing, close and reopen Command Prompt, then try again.

⚠️ Common Error #2: "No such file or directory"

What you see:

pyinstaller: error: argument EXE: No such file or directory: 'myprogram.py'

Solution:

1. Make sure you are in the correct folder

2. Type dir to see all files in current folder

3. Verify your .py file is there

4. Check the spelling of your filename

⚠️ Remember: Command Prompt must be open in the same folder as your .py file!

⚠️ Common Error #3: .exe Opens and Closes Quickly

Problem: You double-click the .exe, a window flashes, then disappears

Solution 1: Add this line at the end of your Python script:

input("Press Enter to exit...")

Solution 2: Run from Command Prompt to see the error:

cd dist
myprogram.exe
💡 The program closes because it finished running. The input() line keeps it open until you press a key!

⚠️ Common Error #4: Antivirus Blocks the .exe

Problem: Antivirus deletes or blocks your .exe file

Solutions:

1️⃣ Click "Run Anyway" if Windows shows a warning

2️⃣ Add the folder to antivirus exceptions

3️⃣ Temporarily disable antivirus (then re-enable)

4️⃣ Use --onefile which is less likely to be flagged

⚠️ Important: This happens because antivirus doesn't recognize new .exe files. Your file is safe if you created it yourself!
